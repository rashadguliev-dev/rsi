# rsi

Ты — элитный quant-исследователь, algorithmic trader и специалист по microstructure рынка Binance.
Ты работаешь на уровне hedge-fund и отвечаешь за конечный результат лично.

Перед тобой код моего индикатора:
MTF Ultimate RSI + EMA crossover (Pine Script).

❗ ВАЖНО:
Тебе ЗАПРЕЩЕНО использовать готовые шаблоны, стандартные «известные» формулы и поверхностные решения.
Твоя задача — САМОСТОЯТЕЛЬНО ВЫВЕСТИ формулы, пороги и правила на основе данных.

────────────────────────────
БАЗОВЫЕ УСЛОВИЯ (BINANCE)
────────────────────────────
1. Использовать ТОЛЬКО реальную модель работы Binance:
   - комиссии как на Binance (spot / futures)
   - maker / taker
   - slippage
   - влияние частоты сделок
2. Любой торговый сигнал считается ДОПУСТИМЫМ ТОЛЬКО если:
   - статистически ожидаемое движение цены ≥ 0.2%
   - иначе сигнал ОБЯЗАН быть отфильтрован

❌ Сигналы «ради движения» запрещены  
✅ Разрешены только сигналы с PROFIT AFTER COSTS ≥ 0.2%

────────────────────────────
ДАННЫЕ
────────────────────────────
- Таймфреймы: 15m / 5m / 1m
- История: минимум 1 ГОД
- Биржа: Binance
- Запрещена подгонка под один участок рынка

────────────────────────────
ЭТАП 1. ГЛУБОКИЙ АНАЛИЗ ИНДИКАТОРА
────────────────────────────
1. Полностью разложи текущую логику RSI + EMA + MTF:
   - где и почему он даёт много сигналов
   - почему именно в боковике он теряет деньги
2. Найди структурную причину:
   - ПОЧЕМУ индикатор входит там, где рынок не способен заплатить ≥0.2%

────────────────────────────
ЭТАП 2. ВЫВОД ФОРМУЛЫ ЭКОНОМИЧЕСКОЙ ПРИГОДНОСТИ
────────────────────────────
ТВОЯ КЛЮЧЕВАЯ ЗАДАЧА:

👉 САМОСТОЯТЕЛЬНО ВЫВЕСТИ ФОРМУЛУ / УСЛОВИЕ, которое отвечает на вопрос:
«Способен ли рынок В ЭТОТ МОМЕНТ заплатить ≥ 0.2% чистого движения?»

Требования:
- формула должна быть основана на данных
- она должна работать ДО входа
- она должна быть переносима в индикатор
- если формула не существует — ты обязан это доказать

────────────────────────────
ЭТАП 3. РЕЖИМЫ РЫНКА
────────────────────────────
Без подсказок с моей стороны:
1. Самостоятельно классифицируй рынок:
   - тренд
   - боковик
   - шум
   - переходные состояния
2. Докажи:
   - в каких режимах достижение ≥0.2% статистически невозможно
   - в каких режимах RSI должен быть ПОЛНОСТЬЮ ОТКЛЮЧЁН

────────────────────────────
ЭТАП 4. ВРЕМЕННАЯ СТРУКТУРА BINANCE
────────────────────────────
Самостоятельно найди:
- часы, где рынок НЕ платит ≥0.2%
- минуты и дни с систематическим шумом
- периоды, где RSI теряет edge из-за microstructure

👉 Итог: логика временного запрета сигналов

────────────────────────────
ЭТАП 5. ML / AI (ЕСЛИ И ТОЛЬКО ЕСЛИ НУЖНО)
────────────────────────────

- результат — интерпретируемые признаки и правила

Если ML НЕ даёт edge:
- докажи, почему он бесполезен в этой задаче

────────────────────────────
ЭТАП 6. ОПТИМИЗАЦИЯ (БЕЗ ПЕРЕОПТИМИЗАЦИИ)
────────────────────────────
Оптимизируй параметры RSI и EMA ТОЛЬКО если:
- это увеличивает долю сделок ≥0.2%
- это устойчиво на разных участках данных

Количество сделок НЕ является целью.

ЭТАП 7. ФИНАЛЬНАЯ ЛОГИКА НОВОГО ИНДИКАТОРА
────────────────────────────

   → «Рынок способен / не способен заплатить ≥0.2%»
2. Условия, при которых RSI:
   - разрешён
   - запрещён
3. Полную анти-боковик логику
4. Логику, пригодную для прямой реализации в Pine Script
